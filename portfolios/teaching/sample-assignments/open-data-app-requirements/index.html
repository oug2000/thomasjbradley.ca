Open Data App Requirements

Assignment requirements for the Open Data App project from MTM 1531 Database and Server-Side and MTM 1526 Rich Media Services.

See more sample assignments…

Overview: Open Data App

Create a small open data map application using Ottawa’s open data sets. You will be continuously developing this app throughout the rest of the term.

Your app’s repository must be hosted on GitHub and your functional app hosted on PHPFog.

Your app will have two public screens:

  1. a map screen that shows all the places from your dataset,
  2. and a single screen that shows information about each place.

Behind the public facing app, you will also create a password protected admin section where you can add, edit, and delete places.

Choose one of the following data sets available on Ottawa’s website:

  • Ball Diamonds 2009
  • Basket Ball Courts 2009
  • Community Gardens
  • Lawn Bowling
  • Museums 2010
  • Outdoor Rinks 2009
  • Skateboard Parks 2009
  • Sledding Hills 2010
  • Splash Pads 2009
  • Sports Fields 2009
  • Tennis Courts 2009
  • Volleyball Courts 2009
  • Wading Pools 2009

The project is split between MTM 1526 and MTM 1531 and composed of many smaller deliverables due throughout the term.

Part 1: Project brief

Create a project brief for your Open Data app—for your chosen data set. It must include the following information:

  1. Description & feature list (in your own words—not copied and pasted from above)
  2. Link to your Github repository
  3. Link to your PHPFog project
  4. Link to your chosen data set
  5. 1 user persona
  6. Sitemap
  7. Wireframe
  8. Design mockup

Your GitHub repository should be named ‘open-data-app’.

Sign up for a PHPFog account. On the second screen choose Custom App under Frameworks. Create a password for MySQL on PHPFog and don’t forget it!. Name your app whatever you’d like.

Mark break down

  • 5—design mockup
  • 4—sitemap
  • 4—wireframe
  • 3—description, features & links
  • 3—user persona
  • 2—HTML, CSS
  • 2—appropriate validation, source code formatting & proper indentation

23 points total

Handing in

Your project brief must be a single, nicely designed HTML page uploaded to Edumedia.

http://imm.edumedia.ca/aaaa0000/open-data-app/

Part 2: Prototype

Using PHP and MySQL, create a basic prototype for your open data application. Your prototype should have the following features:

  • Fake data in your MySQL database.
  • A page that lists all the places.
  • A single page with more details for individual places.
  • An admin section where you can add, edit, and delete places. (Your admin section will be publicly accessible right now.)
  • Admin section should contain all the appropriate validation.

CSS and Javascript are not required for the prototype.

Database

Your database should hold the following information about each place:

  • Name,
  • Longitude,
  • Latitude,
  • and Street Address.

Git & Git commits

Make sure to use Git appropriately. Commit regularly—whenever a single feature has been added or whenever a single bug has been fixed.

Your commit messages must describe exactly what you did. Messages like ‘commit’ or ‘bug fix’ or ‘asdf’ are not acceptable. Some examples of appropriate commit messages:

  • Bug fix: one of the places was not showing properly.
  • New feature: added the edit functionality for places.

Optional bonus challenges

  • Remove as much duplication as possible by reusing pieces of code with includes—add.php and edit.php have a lot of duplication that can be simplified.

Mark break down

  • 7—git commits & messages
  • 7—admin
  • 5—public index.php & single.php
  • 4—database
  • 2—appropriate validation, source code formatting & proper indentation

25 points total

Handing in

The prototype should be available on GitHub and fully functional on PHPFog.

Part 3: Importing data

Import your Open Data dataset into your application, using the script written in class.

Part 4: Google Maps

Add Google Maps, as demonstrated in class, to your Open Data App. Integration must include:

  • Custom icons
  • Info windows

Part 5: Rating locations

Add a rating system, like the one created in class, to your Open Data application.

Part 6: User authentication

Following the process discussed in class, protect your admin section with user authentication and create a username and password just for you.

Part 7: Documentation

Document your open data application by doing the following:

  • Add DocBlocks to the top of all PHP and Javascript files you wrote. Each DocBlock should include: short description, author, copyright, link, package, and version.
  • Create a proper README.md file on GitHub. The Readme should include: description, author info, link to PHPFog, link to the GitHub Pages project brief, and installation instructions.
  • Add a GitHub Pages branch and move your project brief to the branch.

Mark break down

  • 10—DocBlocks
  • 10—README.md
  • 10—GitHub Pages

30 points total

Optional bonus challenges

  • Add a CHANGELOG.md file to your repository.
  • Choose an appropriate license and add that to your repository.
  • Learn how to document functions with DocBlocks and document all the functions in your Javascript and PHP files.

Part 8: Finalized open data app

Finish up the Open Data application and make it work really well.

Some things to do:

  • testing: make sure every feature works as expected;
  • usability improvements;
  • adding the CSS design;
  • browser testing;
  • splitting repetitive HTML into includes;
  • checking indentation and code cleanliness;
  • more testing.

Optional bonus challenges

  • Make the application responsive and mobile first.

Mark break down

  • 7—working & tested code
  • 6—design
  • 5—browser testing
  • 4—includes
  • 4—cleanliness & indentation
  • 4—spit & polish

30 points total

Bonus 1: Geolocation

As covered in class, include a search field on your Open Data app where users can search their address and get a list of the closest locations. Also use the GeoLocation API to automatically find the closest locations.

Mark break down

  • 3—address search
  • 2—geo-location

5 points total

Bonus 2: Disqus

Research and include the Disqus comment system on your single pages.

Bonus 3: Filtering locations

Add the functionality to allow your users to filter the locations. Some sort examples: alphabetical, highest-to-lowest rated, etc.

Bonus 4: Server optimization

Speed up your application by following what Google Page Speed recommends, including:

  • minifying your CSS,
  • minifying your JS,
  • enabling Gzip,
  • using CSS sprites,
  • etc.

Also, make your URLs prettier by enabling mod_rewrite in your application.

Mark break down

  • 3—minification & spriting
  • 2—.htaccess

5 points total

Bonus 5: Organization and security

Fix some security holes in the application and organize the repository in a more secure manner.

As an example, the import-data.php page is currently wide open, and would allow anybody to re-import all the data into our database.

Update your repository and move some files so they are not accessible on the live application. After you have rearranged your repository, go to PHPFog control panel and set the ‘Base Directory’ to www.
